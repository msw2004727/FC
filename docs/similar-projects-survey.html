📊 執行摘要(Taiwan Sports Edition)

核心發現:台灣運動軟體生態 商業競品密度高(60+ 家)、GitHub 開源稀少(35 個)、5 個垂直本輪未查到本土產品。ToosterX 在「LIFF + 俱樂部 + 活動報名 + 簽到 + EXP」整合上,本輪公開搜尋未找到台灣同類開源專案,但商業 SaaS 圈有 18+ 家專業競品(Sporetrofit、Gymomo、Hikingbook、Velodash、Golface 等)規模都比 ToosterX 大。

🏓 匹克球市場真相(寶藏級缺口)

結論:截至本輪公開搜尋(2026-04-27),未找到以台灣本土匹克球社群為核心、同時具備揪局 / 候補 / 積分 / DUPR 整合的成熟產品✏️ 共識

這是這次調查最有商業價值的單一發現 — 本土工具缺口明確 + 政府背景場館擴張 + 國際賽事熱潮三重利多。

驗證證據(8 條)

  1. GitHub `pickleball+taiwan`:0 個 repo(本輪公開搜尋未找到)
  2. 台灣現有兩個並存組織:CTPA 中華民國匹克球協會(pickleball.org.tw,台北,2016 成立)+ CTPF 中華民國匹克球總會(ctpf.org.tw,高雄,自稱 Global Pickleball Federation 會員)— 兩者為平行組織,皆無自建報名系統 / App✏️ 共識
  3. TPEPA 臺北市匹克球協會:只有 FB 社團,無 App
  4. 大家以為的「Reclub」(reclub.co)✏️ 共識:是非台灣、非香港本土的國際性平台,服務全球 35+ 國家(共同創辦人公開資料顯示有美國關聯),非台灣產品
  5. 9 個台灣匹克球俱樂部:Pickle Day / Pickle Side / Banger / Ahhh / Lunar / 樓下 / 寰宇 / 樂活浮州 / 紫陽公園 — 全部用 LINE 群 / Google Form / TinyBot
  6. 2025-09 新北河岸 8 個匹克球場開幕(全 first-come-first-served,公開頁未見本土專屬預約工具)
  7. 2024 亞洲匹克球賽(台中)+ 2025 CAPA Taiwan Open:都用 ACCUPASS / BeClass 通用平台,沒有匹克球專屬工具
  8. ipickleball.tw(匹克球中毒社群):是 WordPress 部落格(資訊整理),非報名系統

對 ToosterX 的意義:這是「場景清晰、本土工具缺口明確、用戶痛點明確」的垂直市場。玩家當前 friction 是「LINE 群接龍」,ToosterX 只要在 Mini App 內做「匹克球揪局 + DUPR 整合 + 球場預約」,就能用在地化體驗避開 Reclub 這類國際性平台的落差。主要要警戒的對手是 Reclub(國際性平台)。

🎯 5 個明確市場機會(工具缺口 + 用戶痛點明確)

1
匹克球專屬 App / 揪局工具 — 本輪未找到台灣本土成熟工具,2025 新北 8 個新球場,CTPA / CTPF 協會體系與民間球會都缺日常管理工具。
建議:Mini App 內做匹克球揪局 + DUPR 整合 + 球場預約。預估投入 1-2 個月可發布 MVP。
2
業餘三鐵社團 / 隊伍管理 — 台灣本土三鐵報名平台 0 家,只有國際 IP(IRONMAN、Challenge Taiwan)自營。業餘隊伍靠 FB 群組。
建議:套既有俱樂部 + 報名模組,加「游 / 騎 / 跑」三段練習紀錄。
3
自行車車隊 LIFF Mini App — Velodash 是 native app(Taiwan 龍頭),Giant / Bryton / Xplova 各自做硬體 app,本輪未找到成熟 LINE 內車隊管理工具。
建議:套俱樂部 + 揪團 + 路線記錄,搭配自行車新文化基金會的環島認證。
4
高山協作隊伍管理 / 山屋預約一鍵 — Hikingbook 主資料庫,但高山協作隊伍靠 Google Sites + Excel。山屋預約政府網站(hike.taiwan.gov.tw)UX 老舊。
建議:套主辦人 / 委託人 / 簽到模組,參考愛陌生留守平台「24 小時待命 + 自動通知」設計。
5
高爾夫教練媒合 / 揪團 — Golface 主導預約 + 計分,但不做教練媒合 + 揪團(碎片化由各教練個人經營)。
建議:套教育型俱樂部 + 教練評價 + EXP / 成就(Birdie 累積等)。

資料採用原則

新增資料只分兩種:一是能從政府、協會、職業聯賽、產品官網或 App 商店公開頁面交叉引用；二是公開查不到,直接標成「未公開 / 未佐證」。下表的「ToosterX 機會」是依來源做的產品推論,不當作市場事實。

  • A 級來源:運動部全民運動署、各球類協會、職業聯賽官方頁。
  • B 級來源:產品官網、Google Play / App Store、公開報名或賽事頁。
  • C 級來源:具名媒體或整理型資料,僅用於補充趨勢,不單獨當定論。
  • 未公開:公開搜尋未見可驗證數字,例如用戶數、營收、留存、轉換率、私人 LINE 社群規模。
球類總體事實:全民運動署 114 年運動現況調查顯示,國人參與運動人口比例為 83.3%,規律運動人口比例為 35.6%,球類運動占 20.2%,且男性球類運動比例 22.9%、女性 17.4%；報告也點名主要球類運動為籃球及羽球,羽球比例為 9.4%,較 105 年增加 4.2%。來源:全民運動署 114 年運動現況調查成果、i運動平台歷年運動現況調查下載區。

共通功能地圖:球類不該每項重做

共通模組 適用球類 公開依據 ToosterX 實作方向
報名 / 候補 / 簽到 羽球、籃球、足球、排球、桌球、網球、匹克球 PlayOne 有零打缺額與隊長登錄情境；Reclub 明確列 RSVP、自動候補、簽到與付款標記；網球協會排名賽頁列報名、候補、會外賽與退賽流程。 做成通用 registration pipeline:名額、候補、取消、簽到、付款狀態、LINE 通知。
排名 / 積分 / 戰績 網球、桌球、羽球、排球、籃球、匹克球 TPBL、TVL、CTFA、網球協會都有官方戰績/排名/排行頁；Reclub 有 player-to-player rankings、leaderboards 與 DUPR 整合。 抽象為 season standings + player ratings,不要把每種球的規則寫死在 UI。
隊伍 / 球員名單 / 角色 棒壘球、籃球、足球、排球、高爾夫、羽球社團 TPBL、TVL、CPBL、CTFA 官方頁都把球隊與球員/數據作為核心入口；Golface 官網與 Google Play 都列有球隊管理。 沿用 ToosterX team 模組,補隊長、副隊長、教練、裁判、出納、場地管理員等角色。
場地 / 時段 / 桌次或場次 羽球、桌球、網球、匹克球、籃球、足球、排球、高爾夫 PlayOne 以地區、星期、時段、程度篩選；中華民國匹克球協會有找球場入口；Golface 主打 tee time 預約；CTFA/TVL/CPBL 官方賽程皆綁定場地。 建立 venue + court + slot 模型,先支援手動排程,後續再接場館 API。
賽事模板 網球、羽球、桌球、匹克球、排球、籃球、棒壘球 網球協會全國排名賽清楚區分報名、會前賽、會外賽、會內賽；Reclub 提供 round robin、league；TVL/TPBL/CTFA 都公開戰績與賽程。 優先做 round robin、單淘汰、雙淘汰、小組循環加淘汰,並保留球類規則插件。
最終修正版結論:球類市場已有多種既有工具、官方資料系統與社群營運方式；更精準的描述是「本輪未找到成熟本土 LIFF / 開源型日常隊務工具」。羽球已有 PlayOne 這類零打查詢產品,高爾夫已有 Golface,籃球/棒球/排球/足球都有職業或協會端官方資料系統,匹克球也有協會、總會與 Reclub 這種國際工具。台灣球類不缺賽事與社群,缺的是貼近日常隊務、可在 LINE 裡低摩擦使用、跨球類複用的輕量管理工具。

🎯 戰略建議(基於本次台灣運動深掘)

🟢 該做(優先序)

1. 鎖定匹克球賽道(高商業價值,本土工具缺口明確)

2024 亞洲匹克球賽剛在台中辦完、2025 CAPA Taiwan Open、2025-09 新北河岸 8 個球場開幕、本輪公開搜尋未找到台灣本土匹克球專屬 App、CTPA / CTPF 雙協會皆無自建系統、9 個球會用 LINE 群。**先佔位先贏,主要警戒國際平台 Reclub**。

2. 切入 4 個明確工具缺口垂直
  • 業餘三鐵社團 / 隊伍管理(0 個本土平台)
  • 自行車車隊 LIFF Mini App(Velodash 是 native,LINE 內工具缺口明確)
  • 高山協作隊伍管理(用 Google Sites + Excel)
  • 高爾夫教練媒合 / 揪團(Golface 不做這塊)

這 4 個都有「小團體頻繁聚會」共通需求,ToosterX 既有功能直接可套。

3. 大學運動社 / 系隊作為早期推廣切入點

GitHub 上 Taiwan 運動 OSS 主力是大學學生(NTU 5+、NCTU/NYCU 2、NTUE 2、NCYU 1),這群人最懂工程、最常自己造輪子,**理解 developer-friendly 平台價值**。系隊每年換主辦人重做工具是結構性痛點。

4. 維持 source-available + README 聲明保留所有權

已加 README.md 做這件事。商業化路線下這是正確選擇,等同 Sentry / PlanetScale / Cal.com 早期策略。

🔴 不該打的戰場

1. 路跑賽事報名(8 家紅海 + 計時硬體 + 物流綁定)

樂活、瀚世比、JoinNow、全統、iRunner、EventPal、U-Pro、馬拉松世界 — 從報名到計時到物流到 LINE Pay 全套完整,正面打很難贏。可走差異化(跑團週訓管理),但別碰賽事報名。

2. 通用活動票券(ACCUPASS / KKTIX 已大)

不要試圖做通用平台。守住「運動 vertical」這個定位才有護城河。

3. 國民運動中心 SaaS(Sporetrofit / Gymomo 主導 B2G)

這兩家都是鴻海背景或拿下 80% 市佔,需要政府關係 + IoT 硬體整合,不是純軟體可贏。

4. 大型連鎖健身房自有 App(World Gym / 健身工廠都有)

連鎖品牌會自己做(門禁卡 + 課表 + 會員費),你介入沒空間。
