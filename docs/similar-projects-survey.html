📊 執行摘要(Taiwan Sports Edition)

核心發現:台灣運動軟體生態 商業競品密度高(60+ 家)、GitHub 開源稀少(35 個)、5 個垂直完全空白。ToosterX 在「LIFF + 俱樂部 + 活動報名 + 簽到 + EXP」整合上仍是台灣 OSS 唯一,但商業 SaaS 圈有 18+ 家專業競品(Sporetrofit、Gymomo、Hikingbook、Velodash、Golface 等)規模都比 ToosterX 大。

11 個運動垂直密度排名

  1. 🔴 路跑 / 馬拉松 — 紅海(8+ 本土平台 + 計時硬體 + 物流綁定)
  2. 🟡 羽球 — 商業 3 家(羽球GO / PlayOne / 揪波)+ GitHub OSS 8+ 個(全台灣最熱垂直)
  3. 🟡 高爾夫 — Golface 一家獨大,週邊揪團 / 教練媒合空白
  4. 🟡 登山 — Hikingbook 30 萬用戶主導,但**山屋預約 / 高山協作隊伍管理**空白
  5. 🟡 自行車 — Velodash + Giant + 政府平台,**LIFF 完全空白**
  6. 🟡 棒球 — 商業 0 家,GitHub OSS 4 個(學校 / 業餘聯盟為主)
  7. 🟢 籃球 — 大型商業真空,只有零碎 OSS
  8. 🟢 足球 / 排球 — 商業近真空,OSS 各 2-3 個
  9. 🟢 三鐵 — 本土報名平台 0 個(只有國際 IP 自營),業餘隊伍管理零工具
  10. 💎 匹克球 — 100% 空白(GitHub 0 repos,本土 app 0 個,9 個球會用 LINE 群)
  11. 🟢 校際 / 系盃 — GitHub OSS 13+ 個(NTU 學生主力),但每年重做

3 個必認識的台灣運動 SaaS 龍頭

  • Velodash 飛騎(自行車) — Taiwan 新創 2017,團騎定位 / 路線 / 群組聊天 / 藍牙裝置整合

🏓 匹克球市場真相(寶藏級空白)

結論:台灣 100% 沒有自製匹克球 App / SaaS / GitHub repo

這是這次調查最有商業價值的單一發現 — 競爭真空 + 政府背景場館擴張 + 國際賽事熱潮三重利多。

驗證證據(8 條)

  1. GitHub `pickleball+taiwan`:0 個 repo(完全真空)
  2. CTPA 中華民國匹克球協會(pickleball.org.tw):無報名系統,只有靜態球場列表 + FB 公告
  3. TPEPA 臺北市匹克球協會:只有 FB 社團,無 App
  4. 大家以為的「Reclub」(reclub.co):是香港團隊做的跨國 App,服務港台越,非台灣產品
  5. 9 個台灣匹克球俱樂部:Pickle Day / Pickle Side / Banger / Ahhh / Lunar / 樓下 / 寰宇 / 樂活浮州 / 紫陽公園 — 全部用 LINE 群 / Google Form / TinyBot
  6. 2025-09 新北河岸 8 個匹克球場開幕(全 first-come-first-served,線上預約完全真空)
  7. 2024 亞洲匹克球賽(台中)+ 2025 CAPA Taiwan Open:都用 ACCUPASS / BeClass 通用平台,沒有匹克球專屬工具
  8. ipickleball.tw(匹克球中毒社群):是 WordPress 部落格(資訊整理),非報名系統

市場規模指標

  • CTPA 列了 全台 100+ 球場
  • FB 社團「Taiwan International Pickleball」+「Taiwan Pickleball Forum」總成員上萬人
  • 媒體:Focus Taiwan、freeshiuan 攻略 持續報導
  • 0 個本地工具去服務這個用戶池
對 ToosterX 的意義:這是「護城河大、競爭真空、用戶痛點明確」的垂直市場。玩家當前 friction 是「LINE 群接龍」,ToosterX 只要在 Mini App 內做「匹克球揪局 + DUPR 整合 + 球場預約」就立刻把 Reclub 在地化體驗超車。唯一要警戒的對手是 Reclub(港團隊),但非本地化就是優勢。

🎯 5 個明確市場機會(空白垂直 + 用戶痛點明確)

1
匹克球專屬 App / 揪局工具 — 100% 競爭真空,2025 新北 8 個新球場,CTPA 協會 + 9 個球會都缺工具。
建議:Mini App 內做匹克球揪局 + DUPR 整合 + 球場預約。預估投入 1-2 個月可發布 MVP。
2
業餘三鐵社團 / 隊伍管理 — 台灣本土三鐵報名平台 0 家,只有國際 IP(IRONMAN、Challenge Taiwan)自營。業餘隊伍靠 FB 群組。
建議:套既有俱樂部 + 報名模組,加「游 / 騎 / 跑」三段練習紀錄。
3
自行車車隊 LIFF Mini App — Velodash 是 native app(Taiwan 龍頭),Giant / Bryton / Xplova 各自做硬體 app,LINE 內車隊管理完全空白。
建議:套俱樂部 + 揪團 + 路線記錄,搭配自行車新文化基金會的環島認證。
4
高山協作隊伍管理 / 山屋預約一鍵 — Hikingbook 主資料庫,但高山協作隊伍靠 Google Sites + Excel。山屋預約政府網站(hike.taiwan.gov.tw)UX 老舊。
建議:套主辦人 / 委託人 / 簽到模組,參考愛陌生留守平台「24 小時待命 + 自動通知」設計。
5
高爾夫教練媒合 / 揪團 — Golface 主導預約 + 計分,但不做教練媒合 + 揪團(碎片化由各教練個人經營)。
建議:套教育型俱樂部 + 教練評價 + EXP / 成就(Birdie 累積等)。

不要試圖做通用平台。守住「運動 vertical」這個定位才有護城河。

這兩家都是鴻海背景或拿下 80% 市佔,需要政府關係 + IoT 硬體整合,不是純軟體可贏。

連鎖品牌會自己做(門禁卡 + 課表 + 會員費),你介入沒空間。
