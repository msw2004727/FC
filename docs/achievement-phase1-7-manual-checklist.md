# 成就 / 徽章 / 稱號 Phase 1~7 手動驗收表

## 使用方式
- 驗收前先清快取或在網址加 `?clear=1`
- 驗收時以正式版資料為主，不以 Demo 模式為準
- 每完成一項，確認「預期結果」完全符合再勾選

## A. 載入與基本結構
- [ ] 開啟首頁後，沒有 `achievement` 相關 console error
  預期結果：站點正常載入，切換到成就頁 / 個人頁 / 後台成就頁都可開啟
- [ ] 切到「成就頁（`page-achievements`）」可以正常顯示卡片
  預期結果：沒有空白頁、沒有按鈕失效、卡片順序與徽章展示正常
- [ ] 切到「成就 / 徽章管理（`page-admin-achievements`）」可以正常顯示列表
  預期結果：列表、編輯、刪除、新增表單都能開啟

## B. 成就頁顯示
- [ ] 已完成成就會顯示在「已完成」區塊
  預期結果：卡片有完成狀態、完成日期不錯位
- [ ] 未完成成就會顯示在「進行中」區塊
  預期結果：進度條與 `current / threshold` 顯示正常
- [ ] 已被淘汰的假條件成就不會出現在前台
  預期結果：看不到 `organize_event`、`earn_badges`、`30d / 7d / 90d / streak` 類成就

## C. 稱號與個人頁
- [ ] 個人頁稱號顯示正常
  預期結果：只會顯示仍然存在且已完成的成就稱號，不會顯示失效舊稱號
- [ ] 稱號頁下拉選單只列出已完成的可裝備稱號
  預期結果：金稱號只來自金成就，一般稱號只來自非金成就
- [ ] 儲存稱號後，個人頁與名片即時同步
  預期結果：個人頁、名片、稱號頁預覽三者一致

## D. 活動統計與成就公式
- [ ] 個人頁的「應到場次 / 完成場次 / 出席率」正確
  預期結果：只算已結束 + 本人 + 有效報名；候補、取消、移除、同行者不混入
- [ ] `attendance_rate` 成就與個人頁出席率一致
  預期結果：達成與否跟個人頁出席率數字完全對得上
- [ ] `complete_event` 只在簽到加簽退後完成
  預期結果：只有簽到但沒簽退時，不應完成此成就
- [ ] `attend_*` 只在實際簽到後累加
  預期結果：只有報名但沒出席時，不應累加出席型成就

## E. 球隊與狀態型成就
- [ ] 入隊審核通過後，只會讓申請者拿到 `join_team`
  預期結果：審核者自己不會被誤判完成 `球隊新人`
- [ ] `reach_exp` / `reach_level` 會依目前用戶資料即時判定
  預期結果：EXP 與等級足夠時，成就會完成；不足時不完成
- [ ] `complete_profile` / `bind_line_notify` 會依實際欄位判定
  預期結果：缺少必要欄位或未綁定 LINE 推播時，不應達成
- [ ] `days_registered` 會依註冊時間判定
  預期結果：修改測試帳號註冊時間後，成就結果會跟著變化

## F. 後台成就管理
- [ ] 新增成就表單的「時間範圍」只剩 `累計`
  預期結果：看不到 `7d / 30d / 90d / streak`
- [ ] 新增成就表單的「動作類型」只剩正式支援條件
  預期結果：看不到 `organize_event`、`list_shop_item`、`sell_shop_item`、`earn_badges`
- [ ] `join_team`、`complete_profile`、`bind_line_notify` 這類布林型條件不會露出錯誤 threshold 邏輯
  預期結果：表單顯示與儲存值合理，不會因多餘 threshold 導致奇怪描述
- [ ] 編輯舊資料時若碰到 legacy 成就，系統會先清理掉
  預期結果：後台列表不殘留假條件成就或 orphan badge

## G. 迴歸驗收
- [ ] 個人頁徽章數、名片徽章、成就頁徽章展示一致
  預期結果：同一個帳號在不同頁面看到的已得徽章數與內容相同
- [ ] 刪除成就後，對應徽章會一起消失
  預期結果：前台、後台都不再看到被刪除成就與其 badge
- [ ] 重新整理頁面後，剛才的資料清理與設定仍然成立
  預期結果：不會因快取或重載把假條件成就帶回來
