# 04 UI Spec

## 建立/編輯表單
### 欄位順序
1. `賽事封面圖片`
2. `賽事名稱`
3. `賽事類型`
4. `主辦俱樂部`
5. `隊伍上限`
6. `比賽日期`
7. `地區`
8. `場地`
9. `報名費`
10. `委託人`
11. `報名開始`
12. `報名截止`
13. `賽事說明`
14. `賽事內容圖片`

### 表單行為
- `賽事封面圖片` 移到最頂
- `賽事類型` 第一階段固定為 `友誼賽`
- `主辦俱樂部` 為必選欄位
- `報名費` 改為：
  - `feeEnabled` 開關
  - 開啟時才顯示 `fee` 金額欄
- `委託人` 保留現有搜尋 + tag 互動

## 賽事詳情頁
### 固定操作按鈕
- `參加賽事`
- `聯繫主辦人`
- `分享賽事`

### 基本資訊區
- 顯示：
  - `主辦俱樂部（俱樂部名（用戶名））`
  - `地區`
  - `場地`
  - `比賽日期`
  - `報名期間`
  - `報名費`
  - `委託人`

## 參加賽事按鈕狀態
### 俱樂部 captain/leader
- 尚未申請：可送出俱樂部申請
- pending：顯示候審中
- approved：可加入自己到 roster
- rejected：顯示已被拒絕

### 一般隊員
- 所屬俱樂部未核准：
  - 按鈕反灰
  - 點擊顯示提示
- 僅一支已核准俱樂部：
  - 直接加入
- 多支已核准俱樂部：
  - 先跳 modal 選隊
- 已在 roster：
  - 顯示可取消參賽

## 參賽隊伍區塊
### 呈現方式
- 每隊一列橫向卡片
- 左側：
  - 俱樂部封面縮圖
  - 俱樂部名
  - 狀態 badge
- 右側：
  - 隊員 roster 清單

### 狀態
- `主辦`
- `已核准`
- `審核中`

### 顯示規則
- `主辦` 與 `已核准` 對所有人可見
- `審核中` 只對主辦/委託/該隊成員可見
- `rejected` 不在公開列表中顯示

## Modal
### 主辦俱樂部選擇 modal
- 建賽時使用
- 僅在使用者有多支可管理俱樂部，或 admin/super_admin 需指定代建俱樂部時出現

### 多隊身份選擇 modal
- 隊員加入 roster 前使用
- 顯示所有已核准且使用者所屬的俱樂部
- 選定後立即加入該隊 roster

## 空狀態與提示
- 尚無已核准隊伍：
  - 顯示只有主辦隊或空白引導
- 一般隊員在未核准前點擊：
  - 提示 `需俱樂部負責人先行報名參賽並經主辦核准後，才可加入名單`
- 名額已滿時主辦核准：
  - 顯示 `賽事隊伍名額已滿，無法再核准`
