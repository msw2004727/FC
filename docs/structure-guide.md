# 系統功能導覽（中文版）

本文件以非技術角度說明系統的整體結構與各功能區塊，適合新加入的團隊成員或非工程背景的人員閱讀。

---

## 一、系統總覽

```
使用者（手機 / 電腦瀏覽器）
    ↓
前端頁面（一個網頁搞定所有功能）
    ↓
┌──────────────────────────────────────────┐
│           功能模組（14 大區塊）              │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ 活動  │ │ 俱樂部  │ │ 賽事  │ │ 個人  │    │
│  └──────┘ └──────┘ └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ 訊息  │ │ 成就  │ │ 掃碼  │ │ 射門  │    │
│  └──────┘ └──────┘ └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ 用戶  │ │ 廣告  │ │ 後台  │ │ 開球  │    │
│  │ 管理  │ │ 管理  │ │ 儀表板│ │ 遊戲  │    │
│  └──────┘ └──────┘ └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐                       │
│  │ 教育  │ │ 養成  │                       │
│  │ 俱樂部│ │ 角色  │                       │
│  └──────┘ └──────┘                       │
└──────────────────────────────────────────┘
    ↓
資料層（讀寫資料庫、處理登入、快取加速）
    ↓
雲端服務（資料庫 + 登入驗證 + 推播通知）
```

**核心概念**：使用者只需開啟一個網頁，所有功能（報名、查看俱樂部、個人資料等）都在同一個畫面中切換，不需另外安裝任何軟體。

## 二、14 大功能區塊說明

### 活動系統（34 個檔案）
**做什麼**：管理所有運動活動的建立、瀏覽、報名、簽到、統計。
**包含功能**：
- 活動列表：首頁活動清單、篩選、時間軸瀏覽、**月曆視圖**（2026-04-22 新增、2026-04-28 改為「運動圖示 x 場次」彙總、上下滑切月、置頂高光）
- 活動詳情：查看活動內容、參加人員、候補名單
- 報名系統：一鍵報名、同行者報名、候補遞補
- 活動建立：建立新活動、設定費用、人數上限、範本
- 活動管理：簽到確認、出席統計、結束或取消活動
- 分享功能：透過通訊軟體分享活動連結給朋友

### 俱樂部系統（16 個檔案）
**做什麼**：讓使用者建立或加入俱樂部，管理隊員名單。
**包含功能**：
- 俱樂部列表：瀏覽所有俱樂部、搜尋特定俱樂部（分頁 + 防抖搜尋 + 全集合搜尋）
- 俱樂部詳情：查看隊員、俱樂部介紹、戰績
- 俱樂部管理：建立俱樂部、編輯資訊、審核入隊申請
- 成員管理：邀請加入、移除成員、角色指派（captainUid / leaderUids / coachUids 純 UID 比對）
- 動態牆：俱樂部成員貼文、留言、按讚（per-team 權限守衛 + 幹部管理）
- 俱樂部分享：透過通訊軟體分享俱樂部邀請連結

### 賽事系統（17 個檔案）
**做什麼**：管理友誼賽、錦標賽等正式比賽的建立與報名。
**包含功能**：
- 賽事列表：瀏覽公開賽事、報名參加（分頁 + 防抖搜尋 + 指紋跳過重繪）
- 友誼賽：俱樂部之間發起友誼賽、審核參賽申請（子集合架構，不再使用內嵌陣列）
- 賽事表單：主辦俱樂部可設定是否參賽；委託人與裁判支援複選
- 球員名單：各隊上場球員登錄與管理
- 賽事通知：建賽、申請、審核等節點自動發送通知
- 賽事分享：透過通訊軟體分享賽事連結

### 個人資料（9 個檔案）
**做什麼**：管理使用者的個人頁面、頭像、暱稱等資訊。
**包含功能**：
- 個人頁面：顯示基本資料、統計數據、成就徽章
- 資料編輯：修改暱稱、頭像、個人簡介
- 名片分享：產生個人名片並分享給朋友

### 訊息系統（9 個檔案）
**做什麼**：站內訊息通知，管理員可群發公告。
**包含功能**：
- 收件匣：檢視收到的通知與系統訊息
- 管理員廣播：向全體或特定用戶發送站內信

### 成就系統（10 個檔案）
**做什麼**：根據使用者的參與行為，自動或手動頒發徽章與稱號。
**包含功能**：
- 成就瀏覽：查看所有可獲得的成就與進度
- 徽章展示：個人頁面展示已獲得的徽章
- 稱號選擇：使用者可選擇顯示的稱號
- 後台管理：管理員建立新成就、上傳徽章圖、手動授予

### 掃碼簽到（5 個檔案）
**做什麼**：活動現場透過手機掃描簽到或簽退。
**包含功能**：
- 掃碼功能：管理員掃描參加者的個人條碼
- 即時通知：被掃碼的人會立即收到簽到確認
- 出席紀錄：自動記錄到場與離場時間

### 射門遊戲（10 個檔案）
**做什麼**：蓄力射門的互動小遊戲，可在網頁上玩。
**包含功能**：
- 遊戲畫面：立體球場場景，拖曳蓄力後射門
- 計分排行：記錄最高分，可查看排行榜
- 實驗室版本：獨立測試頁面，用於遊戲調校

### 用戶管理（4 個檔案）
**做什麼**：管理員查看、管理所有使用者帳號。
**包含功能**：
- 用戶列表：搜尋用戶、查看用戶資料
- 經驗值管理：調整用戶經驗值
- 角色權限：指派管理員或其他角色
- 資料補正：修正用戶資料中的錯誤

### 廣告管理（6 個檔案）
**做什麼**：管理首頁及各頁面上的廣告版位。
**包含功能**：
- 輪播廣告：首頁上方的圖片輪播
- 浮動廣告：頁面邊緣的浮動廣告
- 贊助彈窗：進站時的贊助商彈窗
- 小遊戲廣告：遊戲頁面內的廣告版位

### 後台儀表板（6 個檔案）
**做什麼**：管理員專用的數據總覽與查詢工具。
**包含功能**：
- 數據總覽：活動數、用戶數、報名趨勢等關鍵指標
- 參與查詢：依關鍵字或日期查詢活動參與紀錄
- 報表分享：產生臨時報表連結，分享給其他管理員

### 開球遊戲（6 個檔案）
**做什麼**：另一款立體互動小遊戲 — 開球王。
**包含功能**：
- 遊戲畫面：立體場景中的開球挑戰
- 計分機制：記錄成績，排行榜競爭

## 三、獨立模組說明

除了 14 大功能區塊之外，還有許多獨立模組負責特定功能：

### 首頁相關
| 功能 | 說明 |
|------|------|
| 輪播廣告 | 首頁上方自動輪播的圖片廣告 |
| 系統公告 | 首頁公告欄，顯示最新消息 |
| 彈窗廣告 | 進站時彈出的全屏廣告或公告 |
| 體育新聞 | 首頁每日體育新聞卡片，由雲端自動抓取 |

### 使用者相關
| 功能 | 說明 |
|------|------|
| 角色權限 | 區分一般用戶、管理員、超級管理員等角色 |
| 經驗值自動發放 | 依據參與行為自動累計經驗值 |
| 排行榜 | 依經驗值排名的全站排行榜 |
| 收藏功能 | 使用者可收藏感興趣的活動或俱樂部 |
| 個人儀表板 | 顯示個人參加場次、出席率、經驗值統計 |

### 系統管理相關
| 功能 | 說明 |
|------|------|
| 佈景主題 | 管理員可設定網站深色或淺色主題 |
| 小遊戲管理 | 控制首頁是否顯示小遊戲入口 |
| 資料同步 | 重新計算俱樂部成員數、清理過期資料等維護工具 |
| 報名審計 | 檢查並修復報名資料的不一致問題 |
| 成就批次更新 | 一鍵為所有用戶重新計算成就進度 |

### 日誌與監控
| 功能 | 說明 |
|------|------|
| 日誌中心 | 合併操作日誌、審計日誌、錯誤日誌於同一頁面 |
| 審計日誌 | 記錄管理員的重要操作，供事後查閱 |
| 錯誤日誌 | 記錄系統錯誤並分類嚴重程度 |

### 工具類
| 功能 | 說明 |
|------|------|
| 圖片裁切 | 上傳圖片時可拖曳、縮放、裁切 |
| 圖片上傳 | 將裁切好的圖片上傳至雲端儲存空間 |
| 簽到通知 | 被掃碼簽到時即時收到通知 |

### 商城
| 功能 | 說明 |
|------|------|
| 二手商城 | 刊登、購買、管理二手運動用品 |

### 教育型俱樂部（21 個檔案）
**做什麼**：專為教學型俱樂部設計的學員管理、課程管理與出勤追蹤系統。
**包含功能**：
- 學員管理：學員/家長申請加入、教練審核、自動按年齡性別分組
- 分組管理：建立分組條件、手動或自動歸組、虛擬待審名單
- 課程方案：週期制（固定時段）與堂數制（彈性堂次）兩種模式
- 課程報名：學員報名課程、教練審核、繳費狀態追蹤
- 出勤簽到：群組批次簽到 + QR 掃碼簽到兩種模式
- 出勤紀錄：月曆格子（週期制）或集點卡（堂數制）顯示
- 通知：簽到成功推播、課前提醒、出勤報告

### 養成角色系統（45 個檔案）
**做什麼**：2D 像素風格的寵物養成互動遊戲，可在網頁上玩。
**包含功能**：
- 角色系統：4 個可選角色（白貓、黑貓、藍兔、粉兔），各有獨立精靈動畫
- 體力系統：行動消耗體力、休息恢復，體力不足進入虛弱狀態
- AI 行為：依 MBTI 16 型人格影響行動偏好（追球、攀爬、睡覺等）
- 戰鬥系統：10 種敵人（弓手、法師、劍盾、獸人等），含精英怪
- 場景互動：花朵成長採集、雜草清除、蝴蝶追逐、天氣變化、濃霧事件
- 對話系統：依角色×情緒×MBTI 產生對話氣泡
- 雲端存檔：Firestore 雲端存讀檔 + localStorage 備援

## 四、資料流說明

### 一般瀏覽流程

```
使用者開啟網頁
    ↓
載入基本畫面（使用上次儲存的資料快速顯示）
    ↓
背景連線資料庫，取得最新資料
    ↓
自動更新畫面內容
```

> 這個設計讓使用者即使在網路較慢的環境下，也能在一兩秒內看到畫面，而不是一片空白。

### 報名活動流程

```
使用者點擊「報名」
    ↓
系統檢查名額（是否額滿、是否已報名）
    ↓
寫入資料庫（使用安全鎖定機制，避免超收）
    ↓
重新計算名額，更新畫面
    ↓
若額滿則自動排入候補
```

### 登入流程

```
使用者開啟網頁
    ↓
透過通訊軟體帳號自動登入（不需額外註冊）
    ↓
系統辨識身分，顯示個人化內容
    ↓
管理員會看到額外的管理功能入口
```

### 掃碼簽到流程

```
管理員開啟掃碼功能
    ↓
掃描參加者的個人條碼
    ↓
系統記錄簽到時間
    ↓
參加者手機即時收到簽到通知
```

## 五、雲端服務說明

系統使用以下雲端服務：

| 服務 | 用途 |
|------|------|
| 資料庫 | 儲存所有活動、俱樂部、用戶、報名等資料 |
| 雲端儲存 | 存放圖片（頭像、活動封面、廣告圖等） |
| 雲端函式 | 處理推播通知、新聞抓取、分數提交等後端邏輯 |
| 即時同步 | 資料變更時自動推送到所有使用者的畫面 |
| 通訊軟體登入 | 透過通訊軟體帳號驗證使用者身分 |
