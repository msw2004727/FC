嚴重佈景主題操作

開啟範本編輯器（showTemplateEditor）與儲存全部範本（saveAllTemplates）無權限檢查。範本變更影響全站推播通知。

runAutoExpBackfill() 僅檢查入口權限 admin.auto_exp.entry，無獨立開關。此操作可回溯發放大量經驗值。
現有控制：admin.auto_exp.entry（僅入口） 缺少：admin.auto_exp.backfill
高錯誤日誌清除

現有控制：無（admin.logs.error_delete 未使用） 需要：函式內加入 admin.logs.error_delete 檢查
高站內信撰寫表單

現有控制：admin.messages.entry（僅入口） 需要：函式內加入 admin.messages.compose 檢查

基礎功能保障（所有已登入用戶）

以下 11 項功能不受權限系統控制，任何已登入用戶（包含一般用戶 Lv.0）永遠可以使用。前端不檢查 hasPermission()，後端僅驗證身分（isAuth / isOwner）。

活動瀏覽活動列表與詳情
所有活動列表與單一活動詳情頁為公開資料，任何人（含未登入訪客）皆可瀏覽。
前端：無權限檢查Firestore：events read = true（公開）
活動報名活動
任何已登入用戶可報名公開活動。系統僅驗證身分（userId === 登入者）與活動狀態（open/full）。
前端：無權限檢查CF：registerForEvent — isAuth onlyFirestore：registrations create = isRegistrationOwnerRequest
活動取消報名
用戶可取消自己的報名。系統驗證 userId === 登入者，取消後自動觸發候補遞補。
前端：無權限檢查CF：cancelRegistration — userId === callerUidFirestore：registrations delete = isRegistrationOwnerResource
活動候補排隊
活動額滿時自動進入候補。與報名使用同一個 registerForEvent 流程，系統自動判斷正取或候補。
前端：無權限檢查CF：registerForEvent — 自動候補邏輯
活動取消候補
候補中的用戶可取消候補。與取消報名使用同一個 cancelRegistration 流程。
前端：無權限檢查CF：cancelRegistration — userId === callerUid
俱樂部瀏覽俱樂部與詳情
所有俱樂部列表與詳情頁為公開資料，任何人皆可瀏覽。
前端：無權限檢查Firestore：teams read = true（公開）
俱樂部申請加入俱樂部
任何已登入用戶可提交入隊申請。系統發送站內信給俱樂部幹部審核，24 小時冷卻期防止重複申請。
前端：無權限檢查CF：deliverToInbox — isAuth onlyFirestore：messages create = isAuth + isMessageSenderRequest
俱樂部退出俱樂部
一般成員可自行退出俱樂部（經理/領隊除外，需先轉讓）。系統僅驗證用戶為該俱樂部成員。
前端：無權限檢查Firestore：users update = isOwner + isSafeTeamFieldUpdate
個人管理個人檔案
用戶可編輯自己的暱稱、地區、性別、運動項目等個人資料。
前端：無權限檢查Firestore：users/{uid} update = isOwner + isSafeSelfProfileUpdate
個人收發站內信
用戶可閱讀自己的站內信收件匣，並對系統通知進行操作（同意/拒絕入隊申請等）。
前端：無權限檢查Firestore：users/{uid}/inbox read = isOwner
個人查看個人出席紀錄與成就
用戶可查看自己的活動參與紀錄、出席率、完成場次、以及已獲得的成就徽章。
前端：無權限檢查Firestore：attendanceRecords/activityRecords read = isAuthFirestore：users/{uid}/achievements read = isAuth
